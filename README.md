⚓ Flask Auth & Core Architecture Project🚀 What is this project about? This is a modern Flask web application built with a focus on proper architecture, security, and scalability. Instead of storing all the code in a single file, the project uses a modular structure divided into independent Blueprints.🏗️ What architecture is used? Blueprint pattern: The logic is divided into isolated modules: main (for common pages) and auth (for registration and login). Application Factory (create_app): The application is initialized through a factory, which eliminates the problem of circular imports [app referenced from:]. Secure data storage: The project.db (SQLite) database is moved to an external, isolated instance/ folder, as required by Flask standards.🔒 How are security and authorization structured? Flask-Login: User sessions are managed on the backend. Guests are protected from private routes, and sessions are encrypted with a strong SECRET_KEY key. Password hashing: Clear passwords are never entered into the database. Cryptographic hashing is used via the Werkzeug library (generate_password_hash / check_password_hash). Double data validation: Strict email validation is implemented both on the frontend (HTML5 pattern) and on the backend using regular expressions (re.match by custom EMAIL_REGEX). 🛠️ Technology stack: Backend: Python 3.13+, Flask, Flask-SQLAlchemy, Flask-Login, Werkzeug [app referenced from:]. Database: SQLite (for development), data management and viewing via DBeaver. Frontend: HTML5, Jinja2 (template inheritance via base.html), JavaScript (for dynamic hiding of flash notifications).
